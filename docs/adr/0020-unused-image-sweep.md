# ADR-0020: 미사용 이미지는 스윕으로 정리한다

- 상태: 수락됨
- 날짜: 2026-06-21

## 맥락 (Context)

이미지 업로드는 드롭 즉시 일어나서([ADR-0017](0017-image-upload-storage.md)), 본문에서 이미지를 지우거나 업로드를 취소하면 어느 글도 참조하지 않는 **고아 파일**이 Storage에 남는다. 인라인 GC는 새 글(아직 id 없음)·버려진 드래프트를 못 잡아 지저분하다(ADR-0017).

## 결정 (Decision)

- **스윕** 하나로 정리한다 — 모든 글의 `content`+`cover_image`에서 버킷 이미지 경로를 모아 **참조 집합**을 만들고, 버킷 파일 목록과 대조해 **참조 안 된 파일을 삭제**한다.
- **어드민 버튼**으로 수동 실행(확인 후), 결과(삭제/유지 개수)를 보여준다. 인증 사용자만.

## 이유 (Rationale)

- 한 함수가 **모든 고아**(취소·삭제·버려진 드래프트 업로드 포함)를 잡는다. 글별 추적이 필요 없다.
- 본문 스캔(읽기) + 목록 + 삭제뿐 — **이미지 바이트 전송이 없어** Storage 비용(용량·egress)을 안 늘리고 오히려 용량을 **줄인다**.
- 우선 수동 버튼으로 둔다. 자동(크론)은 나중에 필요하면 같은 액션을 스케줄에 걸면 된다.

## 트레이드오프 (Trade-off)

- **저장 안 한 드래프트의 업로드 이미지**(에디터에만 있고 글로 저장 전)는 참조로 안 잡혀 지워질 수 있다 → **작성 중에는 실행하지 않는다**고 버튼에 안내한다.
- 버킷 목록은 한 번에 최대 1000개로 조회한다(개인 블로그 규모엔 충분). 더 커지면 페이지네이션 필요.

## 결과 (Consequences)

- `features/posts/actions.ts`: `sweepUnusedImages()` Server Action(인증 가드 + 참조 대조 + 삭제).
- `features/posts/components/SweepButton.tsx`: 확인 후 실행·결과 표시. 어드민 대시보드 하단에 배치.
