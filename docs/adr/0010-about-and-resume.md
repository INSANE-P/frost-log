# ADR-0010: 소개와 이력서 페이지

- 상태: 수락됨
- 날짜: 2026-06-18

## 맥락 (Context)

방문자가 "이 블로그 주인이 누구인지" 알 수 있어야 한다. 다만 여긴 블로그지 이력서 사이트가 아니다. 이력서는 아직 준비 중이다.

## 결정 (Decision)

- **소개는 블로그 톤의 자기소개.** 한 문단 인사 + 기술 + 프로젝트 + 활동 + 수상 + 연락처. 학력 같은 이력서용 항목은 뺀다.
- **인물 사진 대신 마스코트(공룡).** 사진은 크롭이 까다롭고, 마스코트가 사이트 톤과 일관된다.
- **프로젝트·활동은 알맹이만 컴팩트하게.** 협업물은 "담당해 개발"처럼 같이 한 사람을 존중하는 표현으로.
- **이력서는 빈 페이지 대신 '준비중'을 마스코트가 톡 튀어나와 알리는 페이지로.**
- 연락처 GitHub은 공용 `GithubIcon`을 쓴다.

## 이유 (Rationale)

- 블로그의 소개는 이력서보다 가벼워야 자연스럽다. 학력·연락처를 전부 나열하면 딱딱하고 이력서 같다.
- 사진은 잘림·톤 관리가 번거롭고, 마스코트(설화 안은 공룡)는 다른 화면과 한 결로 묶인다.
- 협업물에서 "주도"처럼 센 표현은 같이 한 사람들에게 서운할 수 있어 절제한다.
- 준비 중인 걸 빈 화면으로 두면 깨진 듯 보인다. 마스코트로 유쾌하게 안내하면 의도된 상태로 읽힌다.

## 고려한 대안 (Alternatives)

- **이력서 전체를 소개에 그대로** — 정보는 많지만 블로그 톤과 안 맞고 무거워 기각.
- **인물 사진 사용** — 사람 냄새는 나지만 크롭·톤 관리가 번거로워 마스코트로.
- **이력서를 외부 문서로 연결** — 아직 준비 중이라 준비중 페이지로 대체.

## 결과 (Consequences)

- 소개: `src/app/(public)/about/page.tsx`
- 이력서: `src/app/(public)/resume/page.tsx` (마스코트 `dino-pop`)
- 연락처 아이콘: 공용 `GithubIcon`
