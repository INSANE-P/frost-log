# ADR-0006: 레이아웃과 테마 전환

- 상태: 수락됨
- 날짜: 2026-06-17

## 맥락 (Context)

모든 방문자 페이지는 같은 헤더·푸터 위에서 움직인다. 다크 모드 토글은 어디서나 한 자리에 있어야 하고, 모바일에선 좁은 화면에 맞는 메뉴가 필요하다. [ADR-0001](0001-brand-and-logo.md)에서 불꽃을 브랜드 마크로 정했으니, 테마 토글은 그것과 역할이 겹치지 않아야 한다.

## 결정 (Decision)

- **`(public)/layout.tsx`에 헤더+본문+푸터를 공통으로 둔다.** `(admin)`과 경계를 나눈다(ADR-0002).
- **헤더는 sticky.** 로고(불꽃+'설화'), 데스크탑은 인라인 내비, 모바일은 햄버거. 홈은 로고가 대신해 내비에서 제외.
- **모바일 메뉴는 가운데서 얼음판이 퍼지는 풀스크린 오버레이.** 흔한 슬라이드 대신 우리 톤(얼음).
- **테마 토글은 해/달.** 불꽃은 브랜드 전용이라 토글엔 보편적인 해/달을 쓴다(역할 분리).
- **테마 전환은 토글 지점에서 번지듯(켜기)·거둬지듯(끄기).** 모바일 메뉴와 같은 얼음 모티프. View Transitions, 미지원·`reduced-motion`이면 즉시 전환.
- **GitHub 마크는 공용 아이콘으로 추출.** 헤더·푸터·소개가 같은 걸 쓴다.

## 이유 (Rationale)

- 레이아웃을 한 곳에 모으면 페이지마다 헤더·푸터를 반복하지 않고, 이동 경험이 일관된다.
- 토글에 또 불꽃을 쓰면 브랜드 마크와 헷갈린다. 해/달은 누구나 바로 아는 메타포라 직관적이다.
- 전환을 그냥 깜빡 바꾸면 톤이 안 난다. 모바일 메뉴의 얼음 확장과 같은 모티프로 묶으면 사이트가 한 결로 움직인다.
- 같은 아이콘을 여러 곳에 복붙하면 나중에 한 곳만 고쳐 어긋난다. 공용으로 빼 한 번에 관리한다.

## 고려한 대안 (Alternatives)

- **모바일 슬라이드(서랍) 메뉴** — 익숙하지만 개성이 없어 얼음 확장으로.
- **테마 토글을 불꽃 켜기/끄기로** — 브랜드와는 맞지만 무엇을 바꾸는 버튼인지 직관이 약해 기각.
- **전환 없이 즉시 색 교체** — 가볍지만 딱딱해 얼음 번짐을 더했다(저사양·`reduced-motion`은 즉시).

## 결과 (Consequences)

- 레이아웃: `src/app/(public)/layout.tsx`
- 헤더·푸터·모바일·토글: `src/components/layout/*`
- 내비 단일 출처: `src/components/layout/nav.ts`
- 공용 아이콘: `src/components/ui/GithubIcon.tsx`
- 모션: `src/styles/animations.css` (테마 토글, 얼음판 메뉴, 테마 전환)
