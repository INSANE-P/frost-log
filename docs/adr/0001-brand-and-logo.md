# ADR-0001: 브랜드와 로고

- 상태: 수락됨
- 날짜: 2026-06-12

## 맥락 (Context)

개인 블로그의 정체성(이름·테마·마스코트·로고)을 하나의 일관된 컨셉으로 정한다.
개성 있으면서 가독성이 좋고, 오래 머물고 싶은 분위기를 지향한다. 일상·기술·잡담을 아우른다.

## 결정 (Decision)

- **테마명: Frost** — 겨울/서리 세계관.
- **마스코트: 설화(雪火 / 說話)** — "눈 + 불"이자 동음이의로 "이야기".
- **로고: 푸른 불 정령** — 배경 없는 캐릭터(둥근 불꽃 + 큰 눈 + 흰 빛반사). 투명 배경.
- **팔레트: 쿨(서리 블루) 단색 중심.** 라이트/다크 2모드를 CSS 변수 2계층 토큰으로 관리.
- **로고 인터랙션(CSS/SVG):**
  - 불꽃 일렁임 + 눈 깜빡임
  - 마우스를 올리면 촛불처럼 휘며 꺼지고 연기가 남 → 떼면 다시 점화
  - `prefers-reduced-motion` 시 정지

## 이유 (Rationale)

- "차가운 세상에 켜 둔 불"이라는 서사가 "꾸준히 남기는 기록"이라는 블로그 정체성과 맞물린다.
- 푸른 불꽃은 Frost 팔레트와 한 몸으로 일관된 인상을 준다.
- 순수 SVG/CSS라 무한 확대·애니메이션·수정이 자유롭다.
- 2계층 토큰으로 다크모드 전환과 색 교체를 컴포넌트 변경 없이 처리한다.

## 고려한 대안 (Alternatives)

- **추상 눈송이(프리즘/유기적 결정)** — 개성이 약해 기각.
- **여러 캐릭터·형태 후보(정령·동물·횃불 등)** — 작은 사이즈에서의 가독성·일관성에서 단순한 불 정령에 밀림.

## 결과 (Consequences)

- 로고 컴포넌트: `src/components/brand/FlameLogo.tsx` (클라이언트 컴포넌트, CSS 변수로 라이트/다크 자동 대응)
- 파비콘: `public/favicon.svg`
- 토큰·애니메이션: `src/app/globals.css`
- 후속: 마스코트의 다양한 표정/포즈(404·로딩 등)는 별도 이슈에서 확장.
