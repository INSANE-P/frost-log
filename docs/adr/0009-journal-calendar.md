# ADR-0009: 기록 달력과 히트맵

- 상태: 수락됨
- 날짜: 2026-06-18

## 맥락 (Context)

기록은 날짜가 주인공이다(매일의 한 줄). 그래서 이야기 목록과 다른, 날짜를 훑고 고르는 화면이 필요하다.

## 결정 (Decision)

- **연간 히트맵으로 날짜를 훑는다.** 기록 있는 날은 눈송이(각도 제각각), 없는 날은 옅은 자국.
- **히트맵에서 날짜를 누르면 그날 기록만** 아래에 보여주고, 달력 뷰도 그 달로 맞춘다.
- **아래 목록은 목록 뷰 ↔ 달력 뷰 토글**(아이콘만, 군더더기 없이).
- **단색 frost — 주말 색 구분 없음, 박스·그라데이션 없음.** 눈 내린 듯한 톤.
- **기록 상세는 이야기와 같은 레이아웃을 공유**(ADR-0005, `PostArticle`).

## 이유 (Rationale)

- 일기는 "언제"가 핵심이라, 캘린더형으로 훑으면 꾸준함이 한눈에 보이고 빈 날도 부담이 덜하다.
- 흔한 색칠 블록(잔디형) 히트맵은 기계적이고 우리 톤과 안 맞는다. 눈송이로 두면 frost 세계관과 맞고, 각도를 날짜마다 다르게 해 찍어낸 느낌을 피한다.
- 주말을 빨갛게 칠하는 흔한 달력 색은 단색 톤을 깬다. 색은 accent 하나로 절제한다.
- 히트맵(훑기)과 목록/달력(읽기)을 나누면 각자 역할이 분명하다.

## 고려한 대안 (Alternatives)

- **색칠 블록형 잔디 히트맵** — 익숙하지만 기계적이라 눈송이로.
- **기록을 이야기 목록과 같은 행으로** — 날짜 중심성이 약해져 달력형으로 분리.
- **주말 색 구분** — 일반 달력다움은 있으나 단색 톤과 충돌해 기각.

## 결과 (Consequences)

- 달력·히트맵: `src/features/posts/components/JournalCalendar.tsx`
- 기록 목록·상세: `src/app/(public)/journal/page.tsx`, `journal/[slug]/page.tsx`
- 상세는 `PostArticle` 공유(ADR-0005, ADR-0008)
