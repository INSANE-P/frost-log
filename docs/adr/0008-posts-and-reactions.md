# ADR-0008: 이야기 목록·상세와 반응

- 상태: 수락됨
- 날짜: 2026-06-18

## 맥락 (Context)

[ADR-0005](0005-content-model.md)에서 콘텐츠 모델을 정했다. 이제 실제로 읽는 화면 — 목록·상세 — 과, 다 읽은 뒤의 반응을 정한다.

## 결정 (Decision)

- **목록의 태그 필터는 URL 쿼리(`?tag=`)로.** RSC에서 그 값으로 바로 필터한다. 검색·정렬은 글이 쌓이면 추가(지금은 안 만든다).
- **상세는 기록과 같은 레이아웃을 공유한다(ADR-0005).** 제목·요약·바이라인(날짜) → 본문 → 반응 → 이전/다음 → 목록으로.
- **반응은 "불 지피기".** 흔한 하트·좋아요 대신, 브랜드 불꽃을 눌러 켜는 행동으로 둔다. 누르면 한 번 크게 타오르고 다시 누르면 끈다.
- **지금 불 지피기는 낙관적 UI(로컬)만.** 저장·집계는 Supabase 쓰기 연결 이슈에서 붙인다.

## 이유 (Rationale)

- 태그를 쿼리에 두면 링크 공유·새로고침·뒤로가기에서 상태가 안정적이고, 별도 클라이언트 상태가 필요 없다.
- 목록에선 날짜를 숨기고 상세에서만 보여주면, 목록은 제목 위주로 가볍고 상세는 맥락이 충분하다.
- 좋아요를 불꽃으로 바꾸면 "차가운 세상에 불을 지핀다"는 서사(ADR-0001)와 행동이 맞아떨어진다. 작지만 우리다움.
- 읽기 화면이 먼저 필요했고, 반응의 저장은 DB가 붙은 뒤라야 의미가 있어 단계를 나눴다.

## 고려한 대안 (Alternatives)

- **태그 필터를 클라이언트 상태로** — 가능하지만 공유·새로고침에 약해 URL 쿼리로.
- **하트/좋아요 아이콘** — 익숙하지만 개성이 없어 불꽃으로.
- **반응을 지금 DB까지 연결** — 쓰기 경로(DB)가 아직이라 이번 범위에서 제외.

## 결과 (Consequences)

- 목록·상세: `src/app/(public)/posts/page.tsx`, `posts/[slug]/page.tsx`
- 상세 본문: `src/features/posts/components/PostArticle.tsx`
- 반응: `src/features/posts/components/StokeButton.tsx`
- 모션: `src/styles/animations.css` (stoke)
- 후속: 불 지피기 저장·집계(Supabase 쓰기), 본문 Tiptap 렌더.
