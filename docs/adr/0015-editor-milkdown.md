# ADR-0015: 에디터 — Milkdown (WYSIWYG-마크다운)

- 상태: 수락됨
- 날짜: 2026-06-19

## 맥락 (Context)

[ADR-0014](0014-markdown-content.md)에서 본문을 마크다운으로 저장하기로 했다. 그럼 "어떻게 쓸 것인가"가 남는다. 작성자(나)는 raw 마크다운 문법보다 **노션식 작성에 익숙**하다. 개인 블로그에서 가장 중요한 건 작성자가 글을 실제로 즐겁게 쓰는 것이다(안 쓰면 블로그가 죽는다).

## 결정 (Decision)

- **어드민 에디터는 Milkdown(WYSIWYG-마크다운, `@milkdown/crepe`)을 쓴다.** 노션처럼 치면 바로 서식이 보이고, 마크다운 단축키(`## `, `**`, `- ` 등)도 그대로 변환된다.
- **저장·읽기는 그대로 마크다운**(ADR-0014) — 에디터는 마크다운을 import/export만 한다. 읽기 경로(공개 페이지)는 바뀌지 않는다.

## 이유 (Rationale)

- 작성자가 노션에 익숙하므로 raw 마크다운(분할 미리보기)은 마찰이 된다. WYSIWYG가 작성 빈도를 높인다.
- Milkdown은 노션식 단축키 입력을 지원해 **노션 사람도, 마크다운 사람도 편하다.**
- 저장이 마크다운이라 **에디터는 언제든 교체 가능**(종속 위험 낮음). 엔진(ProseMirror) 복잡도는 **어드민 에디터에만** 있고 독자·읽기 경로엔 영향이 없다.
- 마크다운 왕복 엣지케이스는 표준 콘텐츠(제목·굵게·목록·코드·표·이미지)에선 낮다 — 주로 소스 재포맷이라는 미용 문제다. 각주·수식 등은 필요 시 플러그인.

## 고려한 대안 (Alternatives)

- **분할 미리보기(raw 마크다운 textarea + 프리뷰)** — 가장 단순하고 친 그대로 저장(왕복 0)이지만, 노션 습관과 안 맞아 작성 마찰. 작성 즐거움을 우선해 기각.
- **Tiptap / JSON 저장** — WYSIWYG지만 포터블·안전에서 마크다운에 밀려 [ADR-0014](0014-markdown-content.md)에서 이미 기각.

## 결과 (Consequences)

- 에디터: `@milkdown/crepe` — 어드민 전용 클라이언트 컴포넌트(`ssr:false` 동적 로드)
- 본문 저장: 마크다운 text(ADR-0014), 렌더: `react-markdown`(읽기 경로 불변)
- 복잡도는 어드민 에디터에 한정 — 독자·읽기 경로 영향 없음
- 후속: 이미지 업로드(Supabase Storage), 필요 시 각주·수식 플러그인
