# ADR-0007: 홈 큐레이션과 히어로

- 상태: 수락됨
- 날짜: 2026-06-18

## 맥락 (Context)

홈은 방문자가 처음 닿는 곳이다. 내가 누구인지와 무엇을 읽을 수 있는지를 한 화면에서 빠르게 전해야 한다.

## 결정 (Decision)

- **큐레이션 구조** — 명함(히어로) → 골라 둔 이야기(직접 고른 핀) → 최근 기록·최근 이야기(두 줄기).
- **히어로는 명함** — 정적 헤드라인 + 마스코트(설화를 안은 공룡) + 유틸 링크(github·이력서·메일).
- **유틸 링크는 호버 시 글자가 모여 로고로 모핑.** 소개 버튼 대신 링크로 두고 개성을 준다.
- **헤드라인은 정적, 특정 단어만 색으로 강조.** 움직이는 그라데이션은 쓰지 않는다.
- **"전체 보기" 화살표 링크**로 각 흐름 전체로 연결.

## 이유 (Rationale)

- 추천(내가 고른 것)과 최근(자동)을 나누면, 처음 온 사람도 "뭘 먼저 읽지"를 덜 고민한다.
- 기록과 이야기를 두 줄기로 나란히 두면 블로그의 두 성격이 한눈에 보인다.
- 헤드라인이 계속 움직이면 시선이 분산되고 가벼워 보인다. 차분한 헤드라인에 한 단어만 강조하는 게 잘 만든 서비스의 결이고 읽기에 편하다.
- 링크 모핑은 작지만 우리만의 인상을 남긴다. 모션을 못 쓰는 환경에선 색만 바뀌게 해 접근성을 지킨다.

## 고려한 대안 (Alternatives)

- **헤드라인에 움직이는 그라데이션** — 화려하지만 산만하고 톤과 어긋나 기각.
- **추천 없이 최신 글만** — 단순하지만, 처음 온 사람에게 "이걸 먼저 읽어보라"는 안내가 빠져 기각.
- **소개 버튼(CTA)** — 평범해서 유틸 링크 + 모핑으로 대체.

## 결과 (Consequences)

- 홈: `src/app/(public)/page.tsx`
- 히어로 링크: `src/components/brand/MorphLink.tsx`
- 목록 행: `src/features/posts/components/PostRow.tsx`
- 모션: `src/styles/animations.css` (frost-rise, morph-link)
